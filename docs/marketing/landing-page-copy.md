# AI Agent Repo Preflight Kit

Run a deterministic preflight before publishing AI-generated or AI-assisted code.

## The problem

AI coding agents can move faster than your release discipline.

They can leave behind unchecked TODOs, risky public claims, generated artifacts, private-publication surfaces, missing process files, and stale report outputs.

Those problems are not always vulnerabilities. They are release-discipline drift.

## The promise

AI Agent Repo Preflight Kit gives you a local CLI and GitHub Action that scans for obvious release blockers before you merge, demo, publish, or ship.

## What you get

- deterministic local Python CLI
- GitHub Action wrapper
- Markdown report output
- JSON report output
- HTML report output
- SARIF report output
- strict, docs, and public-export profiles
- privacy-first report options
- sample repository and walkthrough
- buyer quickstart and setup guides

## Who it is for

- solo developers using AI coding agents
- technical founders building fast prototypes
- indie hackers preparing public repositories
- engineers publishing templates, tools, or GitHub Actions
- small teams that want a lightweight release preflight

## What it checks

- missing process files
- unchecked release gates
- risky public claims
- secret-bearing filenames
- generated artifacts
- open work and draft-content drift markers
- public-export hygiene issues

## What it is not

This is not a security scanner, compliance scanner, vulnerability scanner, or guarantee of production readiness.

It is a deterministic preflight for visible release-discipline problems.

## One-command demo

```bash
make demo
```

## Buyer outcome

Before publishing, you get a concrete report that tells you whether obvious release blockers are present and where to inspect first.
